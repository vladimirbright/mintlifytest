# Overview

The Launch Checklist is designed to guide you through the process of integrating Truv’s verification services into your Background Screening Platform.

It outlines the key steps for setup, configuration, activation, testing, and post-launch activities to ensure a seamless integration. Use this guide to develop and launch an integration using Truv in your Background Screening Platform.

# Milestone 1: Setup POS

## One-Time Admin Setup:

* Reach out to your Background Screening Platforms' Customer Success Team for the one time admin set up
* Enable Truv as a verification service in the Background Screening Platform of your choice.
* Configure the service and authorize users.
* Input Truv API credentials into the Background Screening Platform.
* Activate the Truv service in Background Screening Platform.

# Milestone 2: Customize & test

## Add Templates in Truv Dashboard

* Customize branding, email, SMS, landing page verbiage.
* Customize number of paystubs and W2 returned.
* Add template for employment history product.
* Configure order expiration settings in Truv Dashboard.
* Set up email notifications for order or status updates.

Refer to the [detailed step-by-step guide to set up templates](/docs/customization-templates) in Truv Dashboard.

## Testing in a Sandbox Environment:

* Input Truv Sandbox API keys, Client ID & Secret
* Create test orders for verification of employment (VOE).
* Verify successful report generation and document upload.
* Ensure Truv Report is generated for all the employers listed for the VOE.

Refer to the [various sandbox credentials](/docs/testing#sandbox-scenarios) based on product type to test in sandbox mode.

## Get Production Access:

* Set up a demo with the Truv team to review implementation and run tests.
* Obtain Truv production API keys and input to your Background Screening Platform.
* Test integration in production with `Truv Sandbox Payroll Provider`

Use the [Testing credentials](/docs/testing#supported-scenarios-and-credentials) to test end-to-end on production.

# Conclusion

Voila! Congrats you can now go-live and launch in production.
